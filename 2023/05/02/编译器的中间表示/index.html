编译器的中间表示

发表于 2023-05-02 更新于 2023-05-03 阅读次数：

前言

这里介绍一下前面编译器设计里说的中间表示相关

中间表示

中间表示IR（Intermediate Representation）是编译器的核心数据结构之一，在传统编译器和机器学习框架里都有极重要的定位。

基本概念

中间表示是编译器表示源代码的数据结构或代码，是介于源语言和目标语言之间的程序表示（应该就是python和机器码）。几乎所有的编译器都需要某种形式的中间表示来对分析、转换和优化的代码进行建模（不然确实没办法做到编译器层面上的优化）。她作为桥梁的性质也要求她必须有足够的表达能力，不丢失信息的情况下能够准确的表示源代码的意思，并且充分考虑到源代码到目标代码的完备性、编译优化的易用性以及性能，总的来说就是要能表示你原本的意思，然后方便进行优化以及不会占用太大的资源

中间表示结构

如图所示引入中间表示的好处：

  1. 它可以面向多个前端，表达多种源程序语言（如是python写的，然后又要求c++也能实现，方便部署），又能对接多个后端，链接不同的目标机器（cpu、gpu、npu等）。
  2. 同时在IR的基础上可以方便的增加更多的优化器，如编译器设计里说的图编译优化、硬件相关的通用编译优化、特定硬件加速IR优化等

优化器

这些优化流程以IR作为输入以IR作为输出，被称为优化器。优化器负责分析并改进IR，极大的提高了编译流程的可拓展性以及降低了优化流程对前端和后端的破坏性（我的理解是如果没有IR和优化器，你进行优化流程的时候前端代码要改很多，然后有的后端硬件又不支持某些优化操作，就会造成代码的不通用）

三个阶段

中间表示的发展经过了三个阶段：

  1. 封闭在编译器里，供编译器的编写者使用，就是谁写编译器谁用，限制就很大，大家只能按开发者的想法来优化
  2. 随着编译器的开源，IR也逐步开源，主要供编译器设计者、分析工具设计者使用
  3. 现在，IR在朝着软件生态构建的方向发展，旨在构建统一的中间表示（比如onnx在部署界的作用）

中间表示的种类

主要有三种吧，线性中间表示 Linear IR、图中间表示 Graphical IR、混合中间表示 Hybrid IR

组织结构 特点 例子
Linear IR 基于线性代码 堆栈机代码、三地址代码
Graphical IR 基于图 抽象语法树、有向无环图、控制流图
Hybrid IR 基于图与线性代码混合 LLVM IR

图中间表示

图中间表示将编译过程的一些信息保存在图中，通过图的对象如节点、边、列表、树等来表示，如ML的模型就是有向无环图，常见的图中间表示有：抽象语法树(Abstract Syntax Tree，AST)、有向无环图(Directed Acyclic Graph，DAG)、控制流图(Control-Flow Graph，CFG)等

混合中间表示

混合中间表示是线性中间表示和图中间表示的结合，这里以LLVM IR [2004LLVM] 为例进行说明。

LLVM(Low Level Virtual Machine)是2000年提出的开源编译器框架项目，旨在为不同的前端后端提供统一的中间表示。

中间表示-LLVMIR

如图所示，LLVM IR使用线性中间表示作为表示的基本块，然后块之间的控制流用图来表示。基本块中以静态单复值指令(Static Single Assignment， SSA)的形式呈现，这些指令构成一个只能怪的先行列表。SSA形式要求每个变量只赋值一次，且变量需要在使用之前定义。控制流图中，每个节点为一个基本块，基本块间通过边实现控制转移。

ps：感觉这个有点像静态图表示，每个算子节点其实都是一系列的线性指令构成的，和算子的参数进行乘加操作之类的，然后算子之间的数据传递用图的边来表示。

机器学习框架里的中间表示

机器学习框架里的IR需要充分考虑以下因素

  1. 张量表示：ml框架处理的数据都是张量，所以正确处理张量是框架的基本要求

  2. 自动微分：网络权重的优化需要求梯度，自动微分是网络模型的自动求导。主流的ml框架都提供了自动微分的功能，在设计中间表示的时候需要考虑自动微分的简洁性、性能以及高阶微分的拓展能力

  3. 计算图模式：主流的框架如Tensorflow、PyTorch、MindSpore都提供了静态图和动态图两种模式：

    • 静态计算图要先创建定义计算图，再显示的执行，有利于对计算图进行优化，高效但不灵活
    • 动态计算图是边运行边构建图，每使用一个算子后，该算子会在计算图中立即执行得到结果，这种方式使用灵活、便于调试，但是运行效率低，中间表示设计同时支持静态图和动态图，可以针对待解决的任务需求，选择合适的模式构建算法模型。

  4. 支持高阶函数和闭包：高阶函数和闭包是函数式编程的重要特性。支持高阶函数和闭包可以抽象通用问题，减少重复代码、提升框架表达的灵活性和简洁性

    • 高阶函数：使用其他函数作为参数，或者返回一个函数作为结果的函数，mindspore里可以输入training step函数到train里面去
    • 闭包：代码块和作用域环境的结合，可以在另一个作用域中调用一个函数的内部函数，并访问到该函数作用域中的成员

  5. 编译优化：ml框架里主要指的是硬件无关的优化、硬件相关的优化、部署推理相关的优化等，这些都依赖于中间表示的实现。

  6. JIT（Just In Time）能力：编译执行加速的时候经常会用到JIT即使编译，JIT编译优化将会对中间表示中的数据流图的可优化部分实施优化，包括循环展开、融合、内联等。中间表示设计是否合理，将会影响机器学习框架的JIT编译性能和程序的运行能力。

PyTorch

PyTorch是一个基于动态计算图的机器学习框架，以python作为前端语言，有很强的易用性和灵活性，方便用户编写和调试网络代码。为了保存和加载网络模型，也提供了TorchScript方法创建可序列化和可优化的模型（类似静态图），TorchScript IR是他的中间表示，通过JIT即时编译的形式，将python代码转换乘目标模型文件，任何ts程序都可以在python进程中保存（就是在训练完后保存），然后加载在别的没有Python依赖的进程中（部署到别的不支持python的机器里）

TorchScript以基于SSA（静态单复值）的线性IR为基本表示，并通过JIT即时编译的Tracing和Scripting两种方法将python转换为TorchScript IR（这里其实也是转onnx的中间过程，转onnx也是换成torchsript再转成onnx的）

TensorFlow

数据流编程

我理解的数据流编程就是构建图，然后数据在图上流动（描述计算任务中数据流和计算之间的依赖关系，节点表示计算操作，边表示的是数据流动），通过将计算任务表示为数据流图，可以方便地进行并行计算和优化。

TensorFlow最初是一个基于数据流编程的机器学习框架，使用数据流图（像onnx在nextron里可视化的那样）作为数据结构进行各种数值计算。在TensorFlow 2.0之前，它主要采用静态图的方式进行计算，即先定义计算图，然后再将数据传入计算图中进行计算。但是，从TensorFlow 2.0开始，它也支持了动态图的方式，即用户可以像使用PyTorch一样进行计算。在动态图中，计算图是动态构建的，每次计算都会重新构建一次计算图。这种方式更加灵活和易于使用，但是在某些情况下可能会牺牲一些性能。
