    前言

    这里简单介绍一下GPT系列的模型

    GPT1

    2018年06月

    Improving Language Understanding by Generative Pre-Training

    Motivation

    视觉里有 ImageNet 这种大规模标好的数据，nlp之前标好的数据量还是比较少，机器翻译有100w的数据，但图片含有的信息比句子多，可能是1：10的信息量，所以要大概1000w级别的数据量

    本文提出了无标注预训练

    之前的工作在finetune的时候构造任务相关的输入，并改一下模型，现在不需要，改下输入就可以了

    困难：

    1、优化函数的怎么选，语言模型、机器翻译、文本一致性用的损失函数都不一样，怎么选？

    2、怎么把学到的文本知识迁移到子任务

    方法

    提出了半监督方法（后续叫自监督）

    1、Transformer or RNN: Transformer 学习到的特征更好，可能是因为Transformer 里有更结构化的记忆来处理更长的文本信息，从而更好地抽取句子层面和段落层面的语义信息

    2、迁移的时候使用的是任务相关的输入表达形式

    Unsupervised pre-training

    对于无监督的序列 \(U = \{u_{1}, u_{2}, ..., u_{n}\}\)，GPT使用语言模型来处理:

    语言模型就是预测下一个词出现的概率

    \[L_{1}(U) = \sum_{i}logP(u_{i}|u_{i-k},...,u_{i-1};\theta)\] 其中k是窗口大小

    \(\theta\)用的是 Transformer 的解码器，即加了 Masked 的 Multi head self attention,如，对U的第i个做预测，先取i前面的 tokens： \[ h_{0} = UW_{e}+W_{p}\\ h_{l} = transformer\_block(h_{l-1}), \forall i\in[1,n]\\ P(u) = softmax(h_{n}W_{e}^{T}) \] 其中\(W_{e}\)是word embedding， \(W_{p}\)是位置编码

    Bert用的是带掩码的语言模型，用的是 encoder 结构，预测的时候既能看见之前的词，也能看到之后的词，Bert的任务比GPT要简单（预测未来和中间插值的区别）

    对于fine-tuning:

    对于给定的序列 \(X = \{x_{1}, x_{2}, ..., x_{m}\}\)，给定标签 y： \[ P(y|x_{1}, x_{2}, ..., x_{m}) = softmax(h_{l}^{m}W_{y})\\ L_{2}(C) = \sum_{(X,y)}logP(y|x_{1}, x_{2}, ..., x_{m}) \] 实际上微调的时候结合上面的任务一起用效果也好，bert 微调的时候只有下游任务的目标函数，没有自监督 loss

    对于分类：

    • 将文本 Text 变成 <start> <text> <Extract>输入到 Transformer 中，并拿 <Extract> 作为提取到的文本语义信息，输入到线性分类层

    对于蕴含（Entailment，给一段话和一个假设，判断这句话有没蕴含假设提出的东西）: - 蕴含的例子：a送了b一个礼物，有三种：a喜欢b，a讨厌b，a和b是邻居，那么结果应该是 True、False、不相关，即做三分类 - 将文本 Text 变成 <start> <premise> <delim> <Hypothesis> <Extract> 输入到 Transformer 中

    对于相似： - 判断两个句子是否相似，相似是对称的，如图所示把f(a,b)+f(b,a)的结果相加进行二分类

    对于多选题： - 多选题是一对多，一个问题，多个答案里选择 - 这里把问题和答案放一起，构造n个序列（n个答案），Transformer 的结果经过线形层计算置信度（0-1），最后使用 softmax

    实验

    GPT:

    BooksCorpus: 7000篇没被发表的书

    模型：12层 Transformer Decoder，ndim=768

    BERT:

    BooksCorpus(800M words),Wikipedia(2500M words)

    BERT_base:12层 Transformer Encoder，ndim=768，encoder 比 decoder 简单一点（少掩码层），110M

    BERT_large:24层 Transformer Encoder，ndim=1024，encoder 比 decoder 简单一点（少掩码层），340M

    GPT2

    2019年2月

    Language Models are Unsupervised Multitask Learners

    • WebText(百万级别的文本)
    • 1B参数
    • Zero shot

    Motivation

    1、目前主流的是在一个任务上收集数据、训练然后做预测（现在模型的泛化性不够好，一个数据集上训练难用到别的数据上）

    2、多任务学习：训练的时候看多个数据集，用多个损失函数，NLP用的不多

    目前 NLP 主流的是大数据集做 pretrain，下游做微调，问题还是有两个：

    1、微调还是得重新训练

    2、需要标记数据

    所以 GPT2 提出了Zero shot，微调的时候不需要标记数据

    方法

    和 GPT-1 模型基本一致，但引入的分隔符、开始符、提取符这些在微调的时候是可以被模型知道具体用来干嘛的，做 ZeroShot 的话由于缺少标注引导，所以不能引入这些标志符号，所以要重构一下输入如把英语翻译成法语时，应该：

    输入：translate to french, english text, french text

    这里 translate to french 起到的就是分割符号的作用，也就是 Prompt

    同理做阅读理解的时候输入为：

    answer the question, document, question, answer

    这样做的原因： - 模型足够强大，能够理解 prompt - 这个结构比较常见，文本数据里面可能已经包含了类似的数据

    数据集

    Common Crawl: 爬虫爬取的各网页数据，TB级别，但是噪声多，清理数据要花很多时间，作者用了其中的 reddit，

    数据集大概800w个文本（40gb）

    模型

    4个变体：

    参数量 layers ndim
    117M 12 768
    345M 24 1024
    762M 36 1280
    1542M 48 1600

    实验

    但是目前和有监督的方法还是差距比较大,但目前没有饱和的趋势

    GPT3

    2020年5月，大力出奇迹

    Language Models are Few-Shot Learners

    GPT1指出，预训练好的模型在使用少数目标样本（few shot）的时候，语言模型能极大的提升性能，GPT2 zero shot 很新但是效果不好，GPT3就是解决效果不好的问题

    GPT3参数量为175 billion，是之前模型的10x倍，所以微调的时候 GPT3 不进行梯度更新

    Motivation:

    基于 GPT2 提出的目前 NLP 主流的是大数据集做 pretrain，下游做微调，问题还是有两个：

    1、微调还是得重新训练

    2、需要子任务相关的标记数据

    3、样本不在数据分布里时，泛化性不见得比小模型好，即微调效果好不一定就是泛化性好，也有可能是过拟合（假设训练集包含了下游任务的一些相关数据，然后对训练集过拟合了，所以在下游效果好，但是在其他的难一点、专业一点的数据效果就不好了），所以设置不允许进行微调的 setting

    4、人类学习不需要大数据集，语言功底好的（提取信息能力强的）给几个例子（few shot）就可以了

    方法

    meta-learning:

    训练一个大模型，存在很多不同任务的样本，训练的时候能够自动学习挖掘很宽广的技巧以及模式识别的能力，推理的时候能很好的应用到想要的任务中

    in-context learning：

    上下文学习，特定上下文环境中学习的机器学习方法。它考虑到文本、语音、图像、视频等数据的上下文环境，以及数据之间的关系和上下文信息的影响。

    如：1. Few shot（示例出现多个，10-100）：6+7=13,6+6=12,5+5=10,8+9=？

    2. One shot（示例出现一个）: 5+5=10,8+9=？

    3. Zero shot（示例没有出现）: 8+9=？

    即在模型预测的时候，分别给出了不同数目的示例来进行预测。我们需要注意的是，这一步骤是没有任何学习的，并没有梯度回传，你可以把它理解成为一个非常详细的prompt，让模型学习到内在的逻辑，进而进行预测。

    所以这里其实是通过few shot，让问题通过让模型处理长序列信息，从而通过注意力机制去自动提取中间的语义信息帮助做下面的事，学习只限于上下文，所以叫上下文学习

    • 优点是不需要新数据了，只需要将问题给模型，让他返回值就行
    • 缺点是假如真的有很多训练数据时不好放进去，另假如有个优质的训练样本，不加训练样本的时候表现不佳，但每次做预测的时候都给这个训练样本做上下文学习，这个重复的过程也不好复用（结果无法保存下来）

    模型变体

    大batch原因：大bs计算性能会好，并行度高，通信成本变低，对分布式好，小模型更容易过拟合，所以bs要变小，导致采样的时候噪音变多

    数据： - Common Crawl，gpt2用的是Reddit里回复大于3的，先训练一个模型，gpt2的数据认为是正样本，common crawl里的是负样本，预测的时候把所有高质量样本保留 - lsh 算法去重（TODO），判断一个集合（文章认为是词的集合）和另一个很大的集合的相似度 - 加入已知的高质量数据（GPT2、GPT、BERT用的数据）

    评估

    采样K个样本，prompt with string "Answer:"或者"A:"

    对于二分类，答案是 True False 而不是0，1，因为训练数据中01出现的概率比较低

    对于QA，和机器翻译一样生成一个序列，采用 Beam Search 去搜索答案

    局限性

    • 长文本生成不太好
    • 结构和算法的局限性，只能往前看，看不到后面
    • 均匀的预测下一个词语，没有说明哪些是重要词语和不重要词（虚词、语气词之类的）
    • 没见过真实物理交互的样子，只见过文本
    • 样本有效性不够（数据太多了）
    • ICL的时候真的是上下文学习么？还是记住了这个样子去记忆里找？
    • 训练耗费大
    • 不可解释
