#include 和 #include "file.h" 的区别

#include 先从标准库的路径中寻找文件，找不到再去工作目录

#include”file.h” 是先从工作目录中寻找文件，找不到再去标准库寻找

C++ 文件编译与执行的四个阶段

1、预处理：根据预处理指令来修改源文件

2、编译：把语言转成汇编

3、汇编：把汇编变成指令集

3、链接：生成可执行程序，包括源程序和机器码的相关描述，如程序多大，要多少内存等

define 和 const 有什么区别

1、处理阶段不同，define是预编译就展开了，const还是和变量一样，在编译的时候才知道值

2、define没有数据类型，const放在静态区

3、define可以定义函数、const不行

4、define不可以用指针指向它，他没有分配内存，const可以

堆和栈有什么区别

堆是手动创建的内存块，栈是变量、函数值这些，如果调用函数 保护断点保护现场的这些情况，

  1. 堆是由 new 分配的内存块，需要程序员手动释放，否则程序结束后会被操作系统自动回收； 栈存放函数的参数值，局部变量，由编译器自动分配释放（保护现场就是用栈）
  2. 堆的分配需要使用频繁的 new/delete ，造成内存空间的不连续，会有大量的碎片（操作系统是用链表来储存空闲的内存地址）
  3. 堆的生长空间向上，地址越来越大，栈的生长空间向下，地址越来越小
  4. 申请栈的空间大小有限制（类比汇编，栈段的大小不能超过 64 kB），申请空间超过栈的剩余空间时，会提示 overflow ，而堆的大小受限于系统中的有效虚拟内存，因此灵活度高，空间也比较大

静态联编和动态联编

编译器通常处理虚函数的方式为：给每个对象添加一个隐藏成员。隐藏成员中保存了一个指向函数地址数组的指针，这种数组称为虚函数表(virtual function table，vtbl) ， 这个表中存储了为类对象进行声明的所有虚函数的地址。e.g. 基类对象包含了一个指针，该指针指向了基类中所有虚函数二点地址表，派生类对象也包含一个指向独立地址表的指针。如果派生类提供了虚函数的额新定义，该虚函数表将保存新函数的地址，否则，该 vtbl 保存的是函数原始版本的地址。如果派生类定义了新的虚函数，那么这个函数的地址也会添加到 vtbl 中。但是注意了，不管类中包含了多少个虚函数，都只需要在对象中添加 1 个地址成员，只是这个成员(数组)的大小不同而已。

说白了就是每个对象会有个虚函数表的地址，

说白了就是防止内存泄漏的东西，比如你在函数里申请了一个指针，然后函数结束的时候，指针变量可能会被销毁，但是指针对象指着的内存可能还在，如果这个函数一直存在下去的话，就容易造成累积占用的内存越来越大，所以智能指针的作用就是当指针销毁的时候（相当于给指针变量一个析构函数？），指针指向的东西也一起销毁掉。
