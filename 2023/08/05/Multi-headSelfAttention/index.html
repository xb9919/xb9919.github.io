Multi-headSelfAttention

发表于 2023-08-05 阅读次数：

前言

最近在回顾Transformer，顺便回顾一下面试高频的multi-head self-attention的两种实现方式

简介

简单的描述一下multi_head self-attention的里流程吧

输入是query、key、mask三个向量，

公式是: \[ Attention(Q,K,V) = softmax(\frac{QK^{T}}{\sqrt{d_{k}}})V \] 拆解就是做以下步骤：

  • 初始化multiheadattention，有4个参数，query_dim, key_dim, embed_dim, num_heads

    • 其中 query_dim是q矩阵的维度，key_dim是k矩阵的维度，embed_dim是要把qk映射到的统一维度（要进行矩阵乘法）,num_heads是头数

  • 初始化3个线性层,W_q, W_k, W_v，把qkv都映射到embed_dim维，这里不用偏置项,一般来说kv矩阵是一样的

  • 输入query[N, T_q, query_dim], key[N, T_k, key_dim], mask[N, T_k],这里T_q 相当于是图像中的 H*W， N 是 batch

  • 先对q,k,v进行特征映射，对于输入的query[N, T_q, query_dim], key[N, T_k, key_dim]分别经过W_q, W_k, W_v， 映射得到query[N, T_q, embed_dim],key[N, T_k, embed_dim],value[N, T_k, embed_dim]

  • 使用torch.split将query、key、value，按第三维进行拆分，分别变成h*[N, T_q, embed_dim/h]再按第0维stack，得到[h, N, T_q, embed_dim/h]

  • 按公式计算attention，先进行q,k矩阵乘法，除以\(\sqrt{embed\_dim}\)得到score[h, N, T_q, T_k]

  • 如果mask不为空，就把转换mask的维度[N, T_k] --> [h, N, T_q, T_k]，然后把score和mask对应为True的位置转为-inf表示不计算attention

    • 这里的mask实际上是调nn.MultiHeadAttention的时候会传入key_padding_mask，这个实际上是每个batch内部的mask(detr有的图片是会经过padding的)，实际上 库还有个attn_mask，应该就是mask掉后面的token的意思，做mask self-attention用的（详情参考这里）
    • 具体是这么做的，先把mask在第1维度和第0维度unsequeeze,变成[1, N, 1, T_k],然后repeate(head, 1, T_q, 1),
      • 为什么这样可行呢？理论上是attention算矩阵乘法嘛，先看后面两个维度，算的是[Tq, dim]@[dim, Tk],T_k是某个图像的像素flatten得到的，矩阵的每一列是像素，其中某些是padding的，即理论上此时的mask是某些列全是True，某些全是False，而矩阵乘法后，和True列相乘得到的位置是列idx和原True列idx一致的元素，而我们进行repeate的时候True列的位置是可以对应的上的（可以简单理解为Tq都是最后一维，对应上了），所以这样做是可行的

  • 对score的结果进行softmax计算得分，并和value相乘得到[h, N, T_q, embed_dim/h]

  • 按head维把score分成head分，得到h*[1, N, T_q, embed_dim/h],再按最后一维concat变成[1, N, T_q, h*T_k]并unsequeeze(0)
