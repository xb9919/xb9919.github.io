最近在复习transformer，碰到了以前的一些概念，还不是理解的很深，这里随缘记录一下

Positional encoding

位置编码信息,一般有两种形式吧，一个是可学习的，一个是AttentionIsAllYouNeed提到的sin、cos形式的， 公式是 \[ PE(pos, 2i) = sin(\frac{pos}{10000^{\frac{2i}{d_{model}}}})\\ PE(pos, 2i+1) = cos(\frac{pos}{10000^{\frac{2i}{d_{model}}}}) \]

这里重点看看后者,初始化只有一个参数，num_pos_feats = Transformer的hidden_dims/2

先上原码：

开始我以为这玩艺儿出来的维度是[b, hidden_dim, h, w]那是像素点的channel的位置编码？实则不然

直接看源码：

self.temperature是公式里的10000，其实这个值算超参吧，用来调节不同位置距离大的时候值应该相差也大

实际上初始化只用到了mask，先对~mask进行行列的cumsum，出来的就是有几行几列了（不含padding的）

然后对其进行归一化（除以最后一行）并x2\(\pi\)转换成角度，此时得到的矩阵y_embed，x_embed就含有了spatial的位置差异信息了

接着算dim_t，是根据公式算的，看着像是对通道dim也做一个差异化，这里想要的是一个channel的位置差异信息

后进行pos_x = x_embed[:, :, :, None] / dim_t和pos_y = y_embed[:, :, :, None] / dim_t，利用的是boardcast机制， 此时pos_x的维度是[b,h,w,hidden_dim/2]， 注意pos_x[0,0,0]和pos_x[0,1,0]是一模一样的，因为x位置差异一样，同理pos_x[0,0,:,0]和pos_x[0,0,:,1]是一样的，因为channel位置差异一样

后面按公式对维度的奇偶性进行sincos，需要注意的是，这里用了个很巧妙的写法：

  • 先分别按dim维取奇数和偶数，并进行stack，对stack不熟悉的可能会理解成concat，想“那奇数维不是变后边去了？”，实际上stack是这样的:

此时再flatten就变成了12121212，确实挺巧妙的

最后再把x方向和y方向concat，那么此时就有了x,y两个方向以及dim的位置信息了
