前言

文件系统相关

统计文件

其中包含三个模块 * ls -l 长列表输出该目录下文件信息(注意这里的文件是指目录、链接、设备文件等)，每一行对应一个文件或目录，ls -lR是列出所有文件，包括子目录。

  • grep "^-" 过滤ls的输出信息，只保留一般文件，只保留目录是grep "^d"。

  • wc -l 统计输出信息的行数，统计结果就是输出信息的行数，一行信息对应一个文件，所以就是文件的个数。

GPU相关

处理僵尸进程

由于训练时用了多线程，程序出现了中断，导致了在某些时候主线程结束了但是子线程还在运行，占用了gpu的空间，要解决这个问题有两种方法：

1、只用一个线程，这个可以从根本上解决，但是单线程训练太慢了

2、出现僵尸进程的时候手动 kill 掉，但是一般僵尸进程通过 nvidia-smi 命令是看不出来的，只能通过 top/htop 或者是 fuser 指令查看

  • 需要注意的是，使用这个命令需要注意权限问题，谁的僵尸进程就谁输入，如果是 root 账户容易影响其他卡（本质上这个指令是清空所有在 nvidia3 卡运行的程序，如果别的程序也有在这会导致子线程被 kill 影响主线程）
